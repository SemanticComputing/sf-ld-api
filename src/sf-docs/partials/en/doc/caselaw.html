Case law

RDF data model

To model the judgments, the ECLI (European Case Law Identifier) metadata schema has been used along with SFCL ontology (Semantic Finlex Case Law), a custom metadata model developed as part of the Semantic Finlex project. SFCL applies FRBR model in a similar manner as ELI and SFL in the case of legislation. However, only one temporal version of each judgment exists and therefore the FRBR model of the SFCL ontology comprises of only the following three classes:

  • sfcl:Judgment (work): judgment as such
  • sfcl:Expression (language version): language version of a judgment
  • sfcl:Format (content format): judgment in a specific content format

URIs

The instructions on how to build URIs for the documents can be found on a separate page.
