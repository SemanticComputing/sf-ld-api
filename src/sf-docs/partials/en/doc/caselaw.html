Case law

RDF data model

To model the judgments ECLI standard (European Case Law Identifier) has been used along with SFCL ontology (Semantic Finlex Case Law), a custom metadata model developed as part of the project. SFCL applies FRBR model in a similar manner as ELI/SFL in the case of legislation. However, judgments do not have different temporal versions and therefore the ontology contains only the following three levels:

  • sfcl:Judgement (work): judgment at an conceptual level
  • sfcl:Expression (language version): language version of a judgment
  • sfcl:Format (content format): judgment in a specific content format

URIs

The instructions on how to build URIs for the documents can be found on a separate page.
