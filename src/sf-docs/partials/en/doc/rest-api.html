REST API (JSON-LD)

The REST API works using redirects and content negotation by using the URI identifiers described in the previous section. Therefore you must allow redirects and set HTTP request Accept header to either application/ld+json or application/json.

Example with curl:

curl -L --header "Accept: application/ld+json" 'http://data.finlex.fi/eli/sd/2004/1430?pretty&tree'

Legislation

List statutes by year (YYYY:

http://data.finlex.fi/eli/sd/(YYYY)

Find a statute by year (YYYY) and id (#) :

http://data.finlex.fi/eli/sd/(YYYY)/(#)

The requests support following parameters:

Parameter Value Description
pointInTime YYYYMMDD get version of a statute / section of a law valid at a given date
language fi, sv language of the content
content txt, html format of the content
pretty - returns JSON in a more readable format
tree - the document content is presented as a hierarchic tree
free - all data returned by the request is available under CC-BY license
  • Examples
    • http://data.finlex.fi/eli/sd/2004/1430?pretty
    • http://data.finlex.fi/eli/sd/2004/1430?pretty&tree

Case law

Find a judgments by court:

http://data.finlex.fi/ecli/(kho|kko)

Find a judgments by year:

http://data.finlex.fi/ecli/(kho|kko)/(YYYY)

Find a judgments by year and court:

http://data.finlex.fi/ecli/(YYYY)

Find a specific judgment by id:

http://data.finlex.fi/ecli/(kho|kko)/(YYYY)/(#)

The requests support following parameters

Parameter Value Description
language fi, sv language of the content
content txt, html format of the content
  • Examples
    • http://data.finlex.fi/ecli/kho/2011/50
    • http://data.finlex.fi/ecli/kho/2011/50?content=html&language=sv
